# Machine Learning Pipeline

## Overview

The machine learning pipeline transforms data generated by the validated Phase 1 computational simulation into a trained XGBoost surrogate model capable of rapid prediction, explainability, optimization, and validation assessment.

The pipeline is fully modular. Each stage is implemented as an independent script that consumes outputs from previous stages without modifying the validated computational model.

## Implemented Stages

| Stage | Script | Primary Output |
| :--- | :--- | :--- |
| **Dataset Generation** | `ml/generate_dataset.py` | `data/simulation_dataset.csv` |
| **Dataset Preprocessing** | `ml/preprocess.py` | `data/processed_dataset.csv` |
| **Train/Test Split** | `ml/split_dataset.py` | Train/Test partitions and `models/preprocessing_pipeline.pkl` |
| **Exploratory Data Analysis** | `ml/eda.py` | `results/eda/` statistical reports and distribution plots |
| **Model Training** | `ml/train_model.py` | Tuned surrogate regressor `models/xgboost_model.pkl` |
| **Model Evaluation** | `ml/evaluate_model.py` | `results/evaluation/` residuals and regression diagnostics |
| **SHAP Explainability** | `ml/shap_analysis.py` | `results/shap/` and `results/result9/` feature attributions |
| **Global Optimization** | `ml/optimize_parameters.py` | `results/result10/10.4 Optimized Parameters.csv` candidates |
| **Validation Feasibility** | `ml/validate_optimization.py` | `results/validation/` feasibility checks and reports |
| **Parameters Panel** | `ml/generate_result7.py` | `results/result7/7.2 Parameter Distribution.png` plot |

## Feature Handling

The preprocessing workflow standardizes the dataset and prepares the machine learning feature matrix.

The trained model uses **only the eight independent formulation and model input parameters**. Derived simulation outputs, including `final_tumor_volume`, `average_drug_concentration`, and `penetration_depth`, are excluded because they are not available when predicting a new formulation and would leak target-related information.

---

## Reproducibility

The pipeline uses fixed random seeds where applicable for:
- Dataset generation
- Train/test splitting
- XGBoost training
- Global optimization

This enables reproducible machine learning experiments.
