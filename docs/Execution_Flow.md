# Pipeline Execution Flow

This document details the step-by-step execution flow of the PLGA nanoparticle mass transport simulation and machine learning pipeline.

---

## 1. Project Phase Overview

The workflow is divided into two major phases:
1. **Phase 1: Computational Simulation**: Solves the core physical mass transport and biological response equations, creating Results 1 to 6.
2. **Phase 2: Machine Learning & AI Pipeline**: LHS dataset sampling, standardizing, training an XGBoost surrogate regressor, feature attribution explainability, global optimization, feasibility checking, and stand-alone weight correlation analysis.

---

## 2. Phase 1: Computational Simulation

Run the primary simulation runner from the repository root:
```bash
python main.py
```
* **Script Actions**:
  - Initializes spatial finite difference parameters ($50 \times 50$ grid) and material properties.
  - Solves the pressure-driven Darcy flow velocity vectors.
  - Computes advective-diffusive mass transport of nanoparticles.
  - Simulates controlled drug release and logistic tumor regression.
  - Executes Results 1-6 studies.
* **Outputs Generated**:
  - Core contour figures: `results/pressure/`, `results/transport/`, `results/drug/`, `results/tumor/`.
  - Result study metrics: `results/result1/` to `results/result6/`.

---

## 3. Phase 2: Machine Learning Pipeline

Execute the machine learning scripts in the chronological sequence shown below. Each module consumes artifacts generated by preceding modules.

### Step 1: LHS Sampling
```bash
python ml/generate_dataset.py
```
- **Actions**: Simulates configurations using Latin Hypercube Sampling.
- **Outputs**: `data/simulation_dataset.csv`.

### Step 2: Preprocessing & Standardization
```bash
python ml/preprocess.py
```
- **Actions**: Cleans dataset, standardizes headers, checks for NaNs/Infs.
- **Outputs**: `data/processed_dataset.csv` and `data/data_quality_report.txt`.

### Step 3: Train/Test Partitions and Scaling Fits
```bash
python ml/split_dataset.py
```
- **Actions**: Creates 80/20 train/test splits, fits scaling pipeline.
- **Outputs**: `data/train.csv`, `data/test.csv`, `models/preprocessing_pipeline.pkl`.

### Step 4: Exploratory Data Analysis
```bash
python ml/eda.py
```
- **Actions**: Plots distributions, correlations, box plots.
- **Outputs**: `results/eda/`.

### Step 5: XGBoost Model Tuning and Training
```bash
python ml/train_model.py
```
- **Actions**: Runs RandomizedSearchCV 5-fold cross-validation.
- **Outputs**: `models/xgboost_model.pkl`, `results/cross_validation_results.csv`.

### Step 6: Test Set Evaluation & Diagnostics
```bash
python ml/evaluate_model.py
```
- **Actions**: Evaluates model, plots residuals, actual vs predicted.
- **Outputs**: `results/evaluation/` and `results/result8/`.

### Step 7: Tree SHAP Explainability
```bash
python ml/shap_analysis.py
```
- **Actions**: Computes SHAP attributions, generates summary and waterfall plots.
- **Outputs**: `results/shap/` and `results/result9/`.

### Step 8: Bounded Parameter Optimization
```bash
python ml/optimize_parameters.py
```
- **Actions**: Searches for top 10 diverse optimal formulations using Differential Evolution.
- **Outputs**: `results/result10/` (including `10.4 Optimized Parameters.csv`).

### Step 9: Feasibility Check
```bash
python ml/validate_optimization.py
```
- **Actions**: Compares model features with independent inputs.
- **Outputs**: `results/validation/` feasibility validation report.

### Step 10: Standalone Correlation Weights
```bash
# Set non-interactive Matplotlib backend on Windows PowerShell
$env:MPLBACKEND="Agg"
python analysis.py
```
- **Actions**: Computes standalone Pearson correlation ranks and weights.
- **Outputs**: `Analysis_Results.xlsx`.
